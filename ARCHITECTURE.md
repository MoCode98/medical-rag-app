INGESTION:
PDF → Parse → Chunk → Embed → Store
                                  │
                                  ▼
                            Vector Database

RETRIEVAL:
Query → Embed → Search → Retrieve Top-K → Format Context
                            │
                            ▼
                    [Chunk 1, Chunk 2, ..., Chunk K]

GENERATION:
Context + Query → LLM → Answer + Citations

┌─────────────────────────────────────────────────────────────────────────┐
│                      SCALABILITY NOTES                                   │
└─────────────────────────────────────────────────────────────────────────┘

Current:
• 10-100 PDFs: Excellent performance
• 1000s of chunks: Fast retrieval (<100ms)
• Local processing: Privacy-preserving

Future Scaling Options:
• Qdrant: Better for 100K+ chunks
• Batch ingestion: Process PDFs in parallel
• Distributed: Multiple Ollama instances
• GPU: Faster embedding/generation
• Caching: LRU cache for frequent queries
